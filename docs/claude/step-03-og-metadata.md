# Step 03 - Open Graph Metadata

## 역할

당신은 Senior Full Stack Engineer입니다.

프로젝트 루트의 아래 문서를 먼저 읽고 작업을 시작해주세요.

- docs/requirements.md
- docs/design-guide.md
- docs/github-info.md
- docs/todo.md

이번 단계에서는 **OG(Open Graph) 메타데이터 자동 수집만 구현**합니다.

---

# 목표

사용자가 URL만 입력해도 메타데이터를 자동으로 가져와
북마크를 더 쉽게 저장할 수 있도록 합니다.

이번 단계에서는 AI는 사용하지 않습니다.

---

# 전제 조건

- Supabase 연동이 완료되어 있습니다.
- CRUD가 실제 DB 기준으로 동작합니다.

---

# 구현 범위

## URL 입력

사용자가 URL을 입력합니다.

예)

https://brunch.co.kr/@xxxx/123

↓

저장 버튼 클릭

↓

메타데이터 자동 수집

↓

폼 자동 채움

↓

저장

---

# 수집 항목

가능하면 아래 정보를 가져와주세요.

- 제목
- 설명
- 썸네일
- 사이트명
- 작성자 (있는 경우)
- 게시일 (있는 경우)

# 저장 방식

사용자가

URL 입력

↓

메타데이터 조회

↓

폼에 자동 입력

↓

사용자는 수정 가능

↓

최종 저장

자동 저장하지 않습니다.

반드시 사용자가 저장 버튼을 눌러야 DB에 저장됩니다.

# 예외 처리

다음 경우를 고려해주세요.

- Open Graph가 없는 사이트
- 접근이 차단된 사이트
- 잘못된 URL
- Timeout

위 경우에도 저장은 가능해야 합니다.
